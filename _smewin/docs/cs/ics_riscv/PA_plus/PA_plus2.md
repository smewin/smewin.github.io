# PA+ Lab2: Optimization

---

作者在 github 上翻阅 nemu 项目时，发现了一个有趣的 PR ，或许可以秒杀这次实验（因为实验都差不多截止了，稍微多说一点应该也问题不大）

https://github.com/NJU-ProjectN/nemu/pull/128

本次实验的关键点，实际就在于轮询和中断的不同处理逻辑。对于一些高请求频率的外设（如时钟），轮询处理外设将会带来巨大的开销，所以对于外设，使用中断处理会大大解放机器的性能。

!!! success "success"
    在你完成了这次 lab 后，你游玩马里奥的体验可能会有了质的变化。如果你实现了声卡，也会发现音乐的播放变得流畅了非常非常多。
    这也让我们明白，一个实现的高效或低效，对于常规的计算类程序而言可能就是多花点时间的事，但对于一个面向用户的服务类程序而言
    ，效率可能是至关重要的。一个程序的效率可能就决定了这个产品的生与死。
